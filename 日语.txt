项目需求文档：智能日语词汇记忆辅助系统 (V1.1)

目标读者： Claude / Codex (AI编程助手)

项目名称： 智能日语词汇记忆辅助系统 (Intelligent Japanese Vocabulary Learning & Memory Aid
System)

项目概述：
本项目旨在开发一个基于人工智能的日语词汇学习辅助系统。该系统将整合背单词、词汇自测、高频词总结和智能记忆效果预测功能，并具备自检机制，以提供个性化、高效且准确的日语词汇学习体验。核心目标是帮助用户更有效地记忆和复习日语词汇，解决传统学习中遗忘快、复习效率低的问题。此外，系统将注重用户界面的美观与易用性，提供定制化的词库导入功能。

1. 项目目标与AI应用点

本项目旨在实现以下关键目标，并着重体现人工智能和数据分析的应用：

1.  个性化学习与定制： 根据用户的学习行为、历史表现和自定义导入的教材，智能推荐词汇和优化复习计划。
2.  记忆效果预测： 利用机器学习模型预测用户对词汇的记忆保持情况。
3.  自适应评估： 自测功能能根据用户表现动态调整题目难度，更精确评估词汇水平。
4.  内容分析： 对大规模日语文本进行分析，提供高频词汇总结。
5.  数据准确性保障： 内置自检机制，确保教学内容的准确性和合理性。
6.  卓越的用户体验： 提供简约、人性化、舒适的UI界面，减少AI工具的冰冷感。

2. 核心功能需求

系统应包含以下核心功能：

2.1. 背单词功能 (Vocabulary Learning Mode)

  - 智能推荐：
    系统根据用户的当前词汇水平、上次复习时间、记忆效果预测（哪些词快要遗忘或需要巩固），智能推荐一批新词或需要复习的旧词。推荐范围将优先考虑用户当前选择或导入的词库。
  - 词库定制与管理（新增）：
      - 导入教材/词库： 用户可以自定义导入外部词汇列表（如CSV/Excel文件），系统应能解析并将其整合到用户的可用词库中。
      - 选择当前词库： 用户可以轻松选择当前要学习的词库（例如：JLPT N3词库、自导入的《大家的日本语》词库、自定义生词本等）。
      - 词库概览： 显示当前词库的词汇总量、已掌握数量、待学习数量等信息。
  - 内容展示：
      - 显示日语词汇（汉字/假名）。
      - 显示假名（平假名/片假名）。
      - 显示中文释义。
      - 显示例句。
      - 显示发音（集成TTS服务）。
      - 显示词性、常用搭配。
  - 用户交互： 提供“认识”、“不认识”、“模糊”等标记按钮。用户的标记数据将作为记忆效果预测模型的输入。

2.2. 自测词汇水平功能 (Self-Assessment Mode)

  - 智能出题：
      - 用户可选择JLPT N5-N1等级或自定义导入词库进行测试。
      - 系统根据用户历史表现，动态调整题目难度，以更精确评估真实水平。
      - 题目类型包括：日译中/中译日、听力识别（播放发音选择词汇）、例句填空、多项选择。
  - 结果分析与反馈：
      - 生成详细报告，包含总分数和在不同难度等级、词性词汇上的掌握情况。
      - 评估用户当前近似的JLPT词汇水平。
      - 基于测试结果，向“背单词功能”推荐接下来重点学习的词汇或难度等级。

2.3. 高频单词总结功能 (High-Frequency Vocabulary Summary)

  - 语料分析：
      - 对大规模日语文本进行分词，统计词频。
      - 分析高频词汇的词性分布。
      - 找出高频出现的词组或固定搭配 (N-gram)。
  - 结果展现：
      - 生成按词频排序的日语词汇列表，可按JLPT等级或特定语料来源筛选。
      - 可视化高频词汇的词云图。
      - 展示常用词汇的固定搭配。
      - 可将高频词汇与用户个人已掌握词汇进行对比，识别并推荐尚未掌握的高频词汇。

2.4. 自检功能 (Self-Correction & Validation)

此功能旨在提升教学内容的准确性和合理性，避免错误教学。

  - 2.4.1. 词汇数据准确性自检 (Data Validation & Correction)
      - 汉字/假名匹配检查： 使用日语分词库（如 MeCab 或 Janome）推断汉字读音，与词汇库中存储的假名比对，不一致则标记为潜在错误。
      - 词性一致性检查： 使用分词库对词汇进行词性分析，与词汇库词性比对，不一致则标记。
      - 释义长度/格式检查： 检查释义是否异常过短或包含非预期字符。
      - 重复词汇检测： 识别词汇库中重复或高度相似的条目。
      - 用户反馈机制： 在背单词和自测界面提供“报错/建议”按钮，收集用户提交的词汇错误或建议，并优先标记待审核。
  - 2.4.2. 教学内容合理性自检 (Teaching Content & Recommendation Logic)
      - 学习曲线异常检测：
        监控用户对特定词汇或难度等级词汇的长期学习表现。若某个词汇在多次复习后仍表现不佳，则标记为“学习异常词汇”，可能暗示词汇数据问题或难度评估有误。
      - 间隔重复推荐合理性检查： 监控用户对推荐复习词汇的交互行为（跳过、延迟、复习后立即遗忘），评估推荐间隔的有效性。
      - 自测题目难度分布检查： 分析用户对不同难度自测题的答题情况。若多数用户对某个特定难度词汇普遍答错/答对，则重新评估其难度标签。

2.5. 用户界面 (UI/UX) 设计 (新增)

  - 简约美观： 界面设计应简洁、清晰，避免视觉上的杂乱，突出核心学习内容。
  - 人性化交互：
      - 直观导航： 各功能模块（背单词、自测、总结、设置）之间切换应自然流畅。
      - 舒适的色彩搭配： 采用柔和、护眼的色彩方案，避免过于鲜艳或刺眼的颜色，减少长时间使用造成的视觉疲劳。
      - 字体与排版： 选择易于阅读的字体，合理调整字号和行距，确保信息传达清晰。
      - 动画与反馈： 在用户操作（如点击按钮、答题对错）时，提供适度的视觉反馈或微动画，增加交互的趣味性和流畅感，避免生硬的AI提示。
      - 无“AI味”： 界面设计应更像一个精心打造的学习工具，而非冰冷的程序界面，强调自然、流畅的人机交互体验。
      - 进度展示： 直观显示学习进度、已掌握词汇数量、今日目标等，提供成就感和激励。

3. 技术栈与数据准备

3.1. 推荐技术栈

  - 编程语言： Python
  - 数据处理： Pandas, NumPy
  - 数据可视化： Matplotlib, Seaborn, Wordcloud (并需考虑Streamlit的内置可视化组件进行更美观的展示)
  - 机器学习： Scikit-learn (Logistic Regression, Decision Tree, Random Forest, SVM)
  - 自然语言处理 (NLP)： MeCab 或 Janome (日语分词、假名推断、词性标注)
  - 文本转语音 (TTS)： gTTS (Google Text-to-Speech Python库) 或集成云服务API (Google
    Cloud/Azure/AWS TTS)
  - 前端/交互界面： Streamlit (强烈推荐，需充分利用其自定义样式和组件，配合CSS/JavaScript实现更高级的UI/UX) 或
    Tkinter/PyQt (桌面应用，同样需要精心设计UI)。
  - 数据存储： SQLite (轻量级数据库，用于存储词汇库、用户学习数据、自定义词库信息、自检日志)。

3.2. 推荐投喂日语资料 (下载地址及用途)

以下是构建系统所需的各类数据源及获取建议：

1.  JLPT词汇表 (核心词汇库)

      - 用途： 构建系统的核心词汇数据库，包含词汇、读音、释义、词性、JLPT等级（作为词汇难度标签）。
      - 获取建议： 搜索并下载已整理好的CSV或Excel文件。
      - 推荐关键词： "JLPT N5 N4 N3 N2 N1 vocabulary list CSV", "JLPT Tango list
        Excel"
      - 参考链接 (请自行验证和寻找最新、最准确的资源)：
          - Github上的日语学习资源库：https://github.com/search?q=jlpt+vocabulary+csv&type=repositories
          - Kaggle数据集：https://www.kaggle.com/datasets?search=jlpt+vocabulary
          - 一些日语学习网站或论坛可能会分享整理好的表格。

2.  日语语料库 (高频词总结与例句)

      - 用途： 进行词频统计以实现“高频单词总结”功能；为词汇提供真实的例句。
      - 获取建议：
          - NHK News Web Easy： 适合作为入门级文本语料，词汇和语法相对简单。可考虑编写爬虫获取其文章内容，但需注意爬虫礼仪和频率。
              - 网址： https://www3.nhk.or.jp/news/easy/
          - 青空文库 (Aozora Bunko)： 提供大量日本文学作品，可用于获取高质量、多样的例句。
              - 网址： https://www.aozora.gr.jp/ (需要爬取其文本内容)
          - 现成的日语词频列表： 有些研究者已对大型语料库进行词频统计，可直接寻找并使用。
              - 搜索关键词： "Japanese word frequency list CSV", "Japanese common
                vocabulary list"
      - 下载地址： 无直接下载链接，需自行爬取或搜索他人整理的公开数据。

3.  模拟学习行为数据 (记忆效果预测模型训练)

      - 用途： 训练和验证“记忆效果预测模型”的核心数据。
      - 获取方式： 需要编写Python脚本自行生成。
      - 数据字段设计：
          - user_id (int)
          - word_id (int, 对应词汇库中的ID)
          - first_study_date (datetime, 首次接触该词汇日期)
          - last_review_date (datetime, 上次复习该词汇日期)
          - total_study_time_minutes (float, 累计学习时长)
          - total_review_count (int, 累计复习次数)
          - last_review_accuracy (float, 0-1, 上次复习掌握程度)
          - interval_days (int, 距离上次复习的天数)
          - memory_outcome (int/float, 预测目标：1=能记住, 0=不能记住; 或0-1的下次测试正确率)。
      - 模拟逻辑：
        结合JLPT词汇难度、艾宾浩斯遗忘曲线原理、间隔重复系统（如SM-2算法）思想来模拟数据。例如，难度越高、间隔越长、学习时长越短，memory_outcome越倾向于“不能记住”。

4.  文本转语音 (TTS) 资源

      - 用途： 为词汇和例句提供发音功能。
      - 获取方式：
          - gTTS Python库： pip install gTTS (在Python环境中直接安装使用)。
          - 云服务API： 前往各自官网注册并获取API密钥。
              - Google Cloud Text-to-Speech:
                https://cloud.google.com/text-to-speech
              - Microsoft Azure Text-to-Speech:
                https://azure.microsoft.com/en-us/services/cognitive-services/text-to-speech/
              - AWS Polly: https://aws.amazon.com/polly/
