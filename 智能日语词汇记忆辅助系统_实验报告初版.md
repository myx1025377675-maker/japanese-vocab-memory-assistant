## 摘要

本项目设计并实现了一个面向日语学习者的“智能日语词汇记忆辅助系统”。系统以日语词汇学习中的高频痛点为出发点，包括遗忘速度快、复习顺序不明确、自测难度不稳定、词库管理不方便以及学习过程缺少数据反馈等问题，综合使用 Python、SQLite、Pandas、Scikit-learn、Matplotlib、WordCloud、Janome 和 Streamlit 等技术，构建了一个可本地运行的智能学习辅助工具。

系统主要功能包括：智能背单词、自适应词汇自测、高频词分析、学习数据分析、词库管理、错题本、自检与数据质量检测、文本转语音、中日互译以及项目说明页。项目中使用 Logistic Regression 和 Random Forest 对词汇记忆保持概率进行建模，并结合 JLPT 难度、复习次数、最近掌握度、平均掌握度和距离上次复习天数等特征计算遗忘风险，从而实现个性化复习推荐。同时，系统会根据自测结果进行加权能力评估，在答题达到一定数量后给出接近 JLPT N5-N1 的词汇水平判断和阶段性学习建议。新增的中日互译功能支持自动识别、日译中和中译日，可用于日语阅读、例句理解和学习材料辅助处理。

与普通词汇表或静态背词程序相比，本系统的特点在于：数据会随着用户交互持续积累，推荐结果、学习分析图表、自测反馈和错题本内容会根据用户行为动态变化；同时，系统支持自定义词库导入和本地 MOJi PDF 词库解析，具有较强的实用性和扩展性。界面方面，系统采用“极简复古杂志风”的 Streamlit UI 设计，使学习工具既具有可读性，也具备较好的展示效果。

## 1. 项目背景与选题意义

### 1.1 背景

日语词汇学习具有数量大、读音复杂、汉字与假名对应关系多样、词义容易混淆等特点。传统的背单词方式通常存在以下问题：

1. 复习顺序固定，不能根据学习者的遗忘风险动态调整。
2. 只记录“是否背过”，缺少对掌握程度和遗忘概率的分析。
3. 自测题目难度过于粗略，容易因为日语汉字与中文意思相似而误判水平。
4. 学习者导入自己的教材或词库比较麻烦，难以形成个性化学习资料。
5. 缺少对长期学习记录的统计、可视化和建议反馈。

因此，本项目希望借助人工智能课程大作业的机会，设计一个既能体现 AI 与数据分析方法，又能真正服务个人日语学习的系统。

### 1.2 选题意义

本系统不是单纯展示算法结果，而是围绕真实学习场景建立了完整闭环：

- 通过背单词行为收集用户学习数据。
- 通过机器学习模型估计词汇记忆保持概率。
- 通过遗忘风险对词汇进行智能推荐。
- 通过自适应自测评估用户词汇水平。
- 通过学习分析图表展示学习状态。
- 通过错题本、自检和反馈机制辅助后续改进。

这种设计体现了人工智能技术在教育辅助场景中的应用价值，也能在课程展示中体现“数据采集、数据处理、模型训练、预测分析、可视化展示、交互式应用”的完整过程。

## 3. 系统总体设计

### 3.1 系统功能模块

当前系统包含以下主要页面：

1. **背单词**
   - 选择当前词库。
   - 根据每日目标和遗忘风险生成推荐队列。
   - 展示单词、假名、释义、词性、JLPT 等级、例句和发音。
   - 用户可点击“认识”“模糊”“不认识”记录掌握程度。
   - 当前词汇可加入错题本。

2. **自测**
   - 随机生成日译中、中译日、假名识别、词形识别等选择题。
   - 同一轮自测中避免重复出现同一单词。
   - 答对后停留一秒并自动进入下一题，答错则停留在当前题目。
   - 达到 8 题后给出初步水平评级。
   - 达到 20 题后生成能力评估和学习建议。
   - 单轮最多 50 题，超过后自动刷新新一轮。

3. **高频词分析**
   - 内置高频词列表，支持查看词汇含义、用法和例句。
   - 支持用户输入日语文本。
   - 使用 Janome 或规则分词进行词频统计。
   - 支持 N-gram 搭配分析。
   - 生成词云图，展示高频词分布。

4. **学习分析**
   - 展示累计学习天数、总复习次数、总自测题数、连续学习天数。
   - 绘制每日学习趋势图、自测正确率图、JLPT 掌握率图、累计掌握词汇增长图。
   - 展示错题词性分布和遗忘风险排行榜。
   - 生成学习画像雷达图和针对性建议。
   - 提供“刷新分析数据”按钮和最近更新时间。

5. **项目说明**
   - 说明系统定位、功能模块、AI 与数据分析方法、数据库设计和大作业要求对应关系。

6. **自检**
   - 检查字段缺失、假名格式异常、重复词条和长期低掌握词。
   - 展示用户反馈记录。
   - 展示记忆预测模型的 Accuracy 和 F1 指标。

7. **词库管理**
   - 查看当前词库词条。
   - 支持重命名已有词库。
   - 支持导入 CSV/Excel 文件并自定义词库名称。
   - 支持删除不需要的词库。

8. **错题本**
   - 查看已加入错题本的词汇。
   - 查看读音、释义、例句和来源词库。
   - 支持移出错题本。

9. **文本转语音**
   - 使用 edge-tts 调用 Microsoft Azure 神经语音。
   - 支持不同日语声线和语速。
   - 支持 MP3 下载。

10. **中日互译**
   - 支持“自动识别”“日语 → 中文”“中文 → 日语”三种方向。
   - 使用免费联网翻译接口，不需要用户配置 API Key。
   - 适合快速查词、短句理解、阅读辅助和写作辅助。
   - 网络或接口不可用时，界面会给出错误提示，避免用户误以为程序无响应。

## 5. 核心算法与人工智能实现

### 5.1 记忆效果预测模型

本项目使用机器学习方法预测用户对词汇的记忆保持概率。由于真实长期学习数据需要较长时间积累，系统先根据学习规律生成模拟学习行为数据，用于训练和验证模型。

模型使用的主要特征包括：

| 特征 | 含义 |
|---|---|
| level_num | JLPT 等级数值化难度，N5 到 N1 分别对应 1 到 5。 |
| review_count | 用户对该词汇的复习次数。 |
| avg_score | 用户历史平均掌握度。 |
| last_score | 最近一次掌握度。 |
| interval_days | 距离上次复习的天数。 |
| days_x_level | 复习间隔与词汇难度的交叉特征。 |

系统训练了两类模型：

1. **Logistic Regression**
   - 优点是模型可解释性较强，适合作为基线模型。

2. **Random Forest**
   - 能处理非线性特征关系，对复习次数、遗忘间隔、词汇难度等因素之间的组合关系更灵活。

系统最终使用模型输出的 `memory_probability` 表示记忆保持概率，并计算：

```text
forget_risk = 1 - memory_probability
```

### 5.2 智能推荐算法

系统在背单词页面使用综合优先级排序生成推荐队列。推荐优先级由以下三部分构成：

```text
priority =
    forget_risk * 0.55
  + interval_days_score * 0.25
  + new_word_score * 0.20
```

其中：

- `forget_risk` 表示模型预测的遗忘风险。
- `interval_days_score` 表示距离上次复习时间越长，越需要复习。
- `new_word_score` 表示复习次数越少的词越需要加入学习队列。

这种设计将机器学习预测与间隔重复思想结合起来，使推荐结果不仅考虑“没学过”，也考虑“快忘了”。

### 5.3 自适应自测与能力评估

自测模块不是简单随机抽题，而是根据历史答题正确率和词汇难度调整抽样权重：

- 答错率较高的词更容易再次被抽到。
- JLPT 难度较高的词具有更高权重。
- 同一轮测试中已经出现的词不会重复出现。
- 题目选项不只包含汉字，也混合假名、词形和释义，减少“日语汉字与中文相似”造成的误判。

当答题数达到 20 题后，系统会计算：

| 指标 | 说明 |
|---|---|
| 普通正确率 | 所有题目的平均正确率。 |
| 加权正确率 | 按 JLPT 难度加权后的正确率。 |
| ability_score | 综合尝试题目难度与答题表现得到的能力分数。 |
| estimated_level | 根据分数估计 N5-N1 词汇水平。 |
| weak_pos | 根据错题统计出的薄弱词性。 |

此外，每个 JLPT 等级都设计了五个阶段的学习建议，并结合名词、动词、形容词等错误频率生成组合建议，使反馈更加个性化。

### 5.4 高频词与文本分析

高频词分析模块体现了自然语言处理和文本分析能力：

- 支持用户输入日语文本。
- 优先使用 Janome 进行日语形态素解析。
- 在 Janome 不可用时使用规则分词作为备选。
- 统计词频并生成词云图。
- 支持 Bigram 和 Trigram 搭配分析。
- 将高频搭配与当前词库掌握情况对比，提示未掌握的高频词。

### 5.5 数据自检机制

系统实现了基础的数据质量检测，包括：

- 字段缺失检查。
- 假名格式检查。
- 重复词条检查。
- 长期低掌握词检测。
- 用户反馈收集。

该模块的意义在于保证学习内容的可靠性，避免词库导入或 PDF 解析带来的错误影响学习效果。

### 5.6 中日互译与阅读辅助

系统新增了“中日互译”模块，用于补充词汇学习以外的阅读和写作辅助能力。该功能支持三种翻译方向：

- 自动识别。
- 日语 → 中文。
- 中文 → 日语。

实现上，系统通过 `翻译助手/translator.py` 中的 `FreeWebTranslator` 调用免费网页翻译接口，包括 MyMemory 和 Google Public Translate 形式的公开接口。用户不需要配置 API Key，也不需要消耗大模型 token。翻译页面使用 Streamlit 表单进行输入，提交后将结果显示在统一的复古杂志风结果框中。

该功能与系统其他模块形成互补：

| 场景 | 作用 |
|---|---|
| 阅读日语材料 | 用户遇到不理解的短句时，可快速获得中文含义。 |
| 日语写作辅助 | 用户可输入中文短句，获得日语表达作为参考。 |
| 背单词例句理解 | 遇到例句难以理解时，可将句子复制到翻译模块辅助理解。 |
| 高频词分析前处理 | 用户可以先理解文本大意，再进行词频和 N-gram 分析。 |

需要说明的是，翻译功能依赖网络和第三方公开接口，因此不作为本系统核心 AI 模型评价指标，而是作为增强实用性的学习辅助模块。

## 6. 系统界面与交互设计

系统采用 Streamlit 实现交互界面，并在默认 Streamlit 样式基础上进行了 CSS 重构。整体风格定位为“极简复古杂志风”，主要特点如下：

1. 使用暖白、浅米色、象牙白作为主背景，减少长时间学习时的视觉疲劳。
2. 使用墨黑、暗红、灰蓝、暗绿等低饱和度颜色，避免现代 SaaS 风格的霓虹渐变。
3. 左侧导航栏设计成类似书籍目录的形式，当前页面用暗红线高亮。
4. 背单词和自测页面采用左右对开布局，类似杂志双页排版。
5. 图表背景与页面纸张色统一，使学习分析页面更像杂志信息图。
6. 按钮、表格、卡片使用细线分隔，减少阴影和大圆角。

这种设计不仅提升了美观度，也让作品在课程展示时更具有完整性和辨识度。

## 8. 实验过程

### 8.1 数据准备

系统内置基础 JLPT 词库和高频词库，并支持从 CSV/Excel 文件导入自定义词库。此外，系统能够自动解析当前目录下的 MOJi PDF 文件，合并为“MOJi N2汉字考前对策”词库。

数据预处理主要包括：

1. 统一词库字段，如单词、假名、释义、词性、JLPT 等级、例句。
2. 对导入文件进行列名映射，兼容 `word/kana/meaning` 和 `单词/假名/释义` 等格式。
3. 对 MOJi PDF 解析结果进行清洗和去重。
4. 对缺失字段、异常假名、重复词条进行自检。

### 8.2 模型训练

系统生成 900 条模拟学习行为数据，用于训练记忆效果预测模型。模拟逻辑考虑了以下因素：

- 词汇难度越高，记忆难度越大。
- 复习次数越多，记忆稳定性越高。
- 最近掌握度越高，预测记住的概率越高。
- 距离上次复习时间越长，遗忘风险越高。

训练流程如下：

1. 生成模拟学习样本。
2. 提取特征并划分训练集、测试集。
3. 分别训练 Logistic Regression 和 Random Forest。
4. 使用 Accuracy 和 F1-score 评估模型。
5. 在系统中使用模型预测词汇记忆保持概率。

### 8.3 系统交互实验

实验过程中，用户可以选择一个词库进行背单词。每次点击“认识”“模糊”“不认识”后，系统会将记录写入 `reviews` 表，并更新已掌握数量、待学习数量和今日建议复习数量。

在自测页面，用户连续回答题目后，系统会将答题结果写入 `test_results` 表。当答题达到 20 题后，系统生成能力评估和学习建议。学习分析页面会读取这些记录并生成图表。

这说明系统的分析结果并非固定展示，而是会随着用户交互数据动态变化。

## 9. 实验结果与分析

### 9.1 背词推荐结果

系统能够根据当前词库和每日学习目标生成推荐队列。例如，当选择 MOJi N2 词库并将今日学习目标设置为 50 时，系统会从当前词库中推荐遗忘风险较高或复习次数较少的词汇。

推荐页面会显示：

- 词汇总量。
- 已掌握数量。
- 待学习数量。
- 今日建议复习数量。
- 当前词汇遗忘风险。

该功能体现了模型预测结果与学习任务安排的结合。

### 9.2 自测分析结果

自测页面能够记录用户的答题情况，并按照题型、JLPT 等级和词性进行分析。与普通测试不同，系统的能力评估不是只看总正确率，而是使用难度加权方法。例如，答对 N2/N1 词汇对能力分数的提升更高，答错高级词也会对评估产生更明显影响。

这种设计能够更合理地估计用户真实词汇水平。

### 9.3 学习分析结果

学习分析页面将用户的背词和自测记录转化为可视化图表，包括：

- 每日学习量趋势。
- 每日自测正确率。
- JLPT 等级掌握率。
- 累计掌握词汇增长。
- 错题词性分布。
- 遗忘风险排行榜。
- 学习画像雷达图。

这些图表能够帮助用户发现学习中的薄弱环节，例如是否学习频率不足、是否某一类词性错误较多、是否高难度词汇掌握不稳定等。

### 9.4 高频词分析结果

高频词分析模块可以从文本中提取常见词汇和词组搭配，帮助用户从真实语料中发现值得优先学习的词。对于日语学习者来说，这比单纯背固定词表更贴近真实阅读场景。

### 9.5 自检结果

自检页面能够帮助发现词库中可能存在的问题，例如：

- 假名格式异常。
- 字段缺失。
- 重复词条。
- 长期低掌握词。

这对于从 PDF、Excel 或网络资料导入词库的系统尤其重要，因为外部数据不可避免会存在格式不统一或识别错误。

### 9.6 中日互译结果

中日互译页面能够对日语短句、中文短句和混合输入进行快速处理。用户可以选择自动识别，也可以手动指定“日语 → 中文”或“中文 → 日语”。该功能适合在学习过程中临时查句子、理解例句或辅助日语表达。

实验中可选用如下示例进行展示：

| 输入内容 | 翻译方向 | 预期展示 |
|---|---|---|
| 今日はいい天気です。 | 日语 → 中文 | 输出对应中文含义。 |
| 我每天复习日语单词。 | 中文 → 日语 | 输出日语表达。 |
| こんにちは | 自动识别 | 自动识别为日语并翻译为中文。 |

该模块增强了系统的学习实用性，使系统不仅能“背词”和“测试”，也能辅助真实阅读和表达。

## 10. 项目加分点与实用性说明

本项目相比普通课程演示程序，具有以下加分点：

1. **真实学习场景明确**
   - 项目来源于作者作为日语学习者的真实需求，不是为了套用算法而临时构造场景。

2. **数据闭环完整**
   - 系统能够采集用户背词、自测、错题和反馈数据，并反过来影响推荐、分析和建议。

3. **机器学习与规则方法结合**
   - 使用 Random Forest 预测记忆保持概率，同时结合间隔重复规则和自测加权规则，使系统更稳定。

4. **支持多来源词库**
   - 系统内置高频词库、JLPT 扩展词库、MOJi PDF 词库，并支持用户导入 CSV/Excel。

5. **自适应评估更细致**
   - 自测不是只给“对/错”统计，而是结合 JLPT 难度、题型和词性错误生成学习建议。

6. **学习分析可视化完整**
   - 系统包含趋势图、掌握率图、词性错题图、遗忘风险排行和雷达图，满足数据分析展示要求。

7. **具备自检与反馈机制**
   - 可检测词库质量问题，并允许用户提交报错或建议。

8. **界面设计有辨识度**
   - 系统采用极简复古杂志风 UI，不是默认 Streamlit 页面，展示效果更完整。

9. **加入中日互译辅助**
   - 支持自动识别、日译中和中译日，适合查词、理解例句和辅助写作，使系统更接近真实日语学习工具。

10. **可长期使用**
   - 数据保存到本地 SQLite，不会因为刷新页面丢失学习记录。

11. **扩展方向清晰**
   - 后续可继续加入听力题、例句填空、学习周报和更真实的长期模型训练。

## 11. 不足与改进方向

虽然系统已经实现了完整的学习闭环，但仍存在一些不足：

1. **真实长期学习数据仍不足**
   - 当前记忆预测模型主要依赖模拟数据和少量真实交互数据，随着后续使用时间增加，可以用真实数据重新训练模型。

2. **PDF 解析结果仍可能存在识别误差**
   - MOJi PDF 中部分假名和释义可能因 PDF 字体编码问题出现识别异常，需要通过自检和人工修正继续优化。

3. **题型还可以更丰富**
   - 当前主要是选择题，后续可加入听力识别、例句填空、输入假名等题型。

4. **TTS 与联网翻译依赖网络环境**
   - 文本转语音功能需要安装 edge-tts，并可能受网络环境影响；中日互译功能使用免费联网接口，接口不可用时翻译可能失败。

5. **模型解释性可以进一步增强**
   - 后续可以加入特征重要性图，让用户和老师更直观看到哪些因素影响遗忘风险。

6. **报告导出功能尚未完成**
   - 后续可支持一键导出学习报告，方便用户长期跟踪学习效果。

## 12. 结论

本项目完成了一个可本地运行的智能日语词汇记忆辅助系统。系统围绕日语学习中的词汇记忆问题，综合实现了词库管理、智能背词、自适应自测、高频词分析、学习数据分析、错题本、自检、文本转语音和中日互译等功能。

在人工智能方法方面，项目使用 Logistic Regression 和 Random Forest 训练记忆效果预测模型，并结合复习间隔、词汇难度和历史掌握度计算遗忘风险，实现智能推荐。在数据分析方面，系统能够根据用户交互记录生成学习趋势、掌握率、错题分布、遗忘风险排行榜和学习画像。在应用价值方面，系统支持多词库、自定义导入、错题复盘、语音辅助和中日互译，具有较强实用性。

总体来看，本系统较好地体现了人工智能课程大作业对算法应用、数据处理、可视化展示和完整交互系统的要求，也具有继续迭代为个人长期学习工具的潜力。
